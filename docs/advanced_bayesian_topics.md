# 発展課題: ベイズ的サイコロ分析の拡張

この資料では、通常の確率推定を一歩進めて、次の4つの発展テーマを扱います。

1. 事前分布の感度分析
2. 事後予測シミュレーション
3. 公平なサイコロとの比較
4. 時間によって歪みが変わるケースの検知

既存の入門デモはそのままにしてあり、以下はすべて**発展課題用の独立デモ**です。

## 1. 事前分布の感度分析
### 何を見るか
観測が少ないとき、事前分布の違いによって予測結果がどの程度変わるかを見ます。観測が増えると、その差は小さくなるはずです。

### 実行コマンド
python prior_sensitivity_study.py

### 読み取り方
- 少数観測では prior ごとの差が大きい
- 多数観測では差が縮む
- ベイズ更新により、データが prior を上書きしていく

## 2. 事後予測シミュレーション
### 何を見るか
現在の事後分布を使って、未来の出目を何度もシミュレーションします。これは「今の知識を前提にすると、どんな未来がどのくらいありそうか」を表します。

### 実行コマンド
python posterior_predictive_study.py

### 読み取り方
- 未来サンプル数が少ないと、見た目のばらつきが大きい
- サンプル数を増やすと、理論的な予測確率に近づきやすい
- 最も有力な面がどの程度安定して現れるかも確認できる

## 3. 公平なサイコロとの比較
### 何を見るか
推定した分布が、公平なサイコロの一様分布 $[1/6, \dots, 1/6]$ からどれくらい離れているかを比較します。

### 実行コマンド
python fairness_comparison_demo.py

### 読み取り方
- MAE や TVD が小さいほど、公平なサイコロに近い
- 偏った乱数源では、これらの値が大きくなる
- 「公平かどうか」を、観測だけから比較できる

## 4. 時間によって歪みが変わるケースの検知
### 何を見るか
前半と後半で乱数源の性質が変化するとき、移動窓ごとに posterior を比較して変化点を探します。

### 実行コマンド
python time_varying_bias_demo.py

### 読み取り方
- ある時点の前後で、最有力面や公平性からの距離が急に変わる
- その差が最大になる場所を、変化点候補として解釈する
- 厳密な推定というより、学習用の直感的な検知法として理解する

## 確認問題
### 問題
1. なぜ観測が少ないほど、事前分布の感度が大きくなりますか。
2. 事後予測シミュレーションは、単なる乱数生成と何が違いますか。
3. 公平なサイコロとの比較で TVD が大きいとは、どういう意味ですか。
4. 時間変化する歪みの検知で、窓ごとの posterior を比べる理由は何ですか。

### 答え
1. データの情報量がまだ少ないため、事前の仮定が推定結果に相対的に強く効くからです。
2. 現時点の観測を反映した事後分布に基づいて未来を予測している点が違います。つまり、学習済みの信念を使ったシミュレーションです。
3. 推定分布が一様分布から大きく離れており、公平とは言いにくい状態を意味します。
4. 同じ乱数源なら隣り合う窓の分布は大きく変わりにくいですが、歪みが変化すると推定分布の差が急に大きくなるためです。
